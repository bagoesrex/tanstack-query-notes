# Dependent Queries

**Dependent queries** (or serial queries) only run **after** another query has finished - useful when the second query needs data from the first.

## Using `useQuery`

Use the `enabled` option to delay execution until conditions are met:

```tsx
const { data: user } = useQuery({
  queryKey: ["user", email],
  queryFn: getUserByEmail,
});

const userId = user?.id;

const { data: projects } = useQuery({
  queryKey: ["projects", userId],
  queryFn: getProjectsByUser,
  enabled: !!userId, // waits until userId exists
});
```

Flow:

- Starts as `isPending` + `fetchStatus: 'idle'`
- Runs when `userId` is available
- Ends as `isSuccess` + `fetchStatus: 'idle'`

## Using `useQueries`

You can also make dynamic dependent queries:

```tsx
const { data: userIds } = useQuery({
  queryKey: ["users"],
  queryFn: getUsersData,
  select: (users) => users.map((u) => u.id),
});

const usersMessages = useQueries({
  queries: userIds
    ? userIds.map((id) => ({
        queryKey: ["messages", id],
        queryFn: () => getMessagesByUsers(id),
      }))
    : [],
});
```

> Returns an **array of query results**.

## Performance Note

Dependent queries create a **request waterfall** → slower loading.
Try combining endpoints (e.g., `getProjectsByUserEmail`) to fetch data **in parallel** instead.
