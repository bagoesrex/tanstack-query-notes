# Disabling or Pausing Queries

If you ever want to disable a query from automatically running, you can use the `enabled = false` option. The enabled option also accepts a callback that returns a boolean.

When you set `enabled: false`:

- The query **does NOT auto-fetch** on mount.
- It does **not** refetch in the background.
- It does **not** respond to `invalidateQueries` or `refetchQueries`.
- If cached data exists → **status: success**.
- If no cached data → **status: pending, fetchStatus: idle**.
- You can still **manually run** `refetch()` (except when using `skipToken`).

```tsx
function Todos() {
  const { isLoading, isError, data, error, refetch, isFetching } = useQuery({
    queryKey: ["todos"],
    queryFn: fetchTodoList,
    enabled: false,
  });

  return (
    <div>
      <button onClick={() => refetch()}>Fetch Todos</button>

      {data ? (
        <ul>
          {data.map((todo) => (
            <li key={todo.id}>{todo.title}</li>
          ))}
        </ul>
      ) : isError ? (
        <span>Error: {error.message}</span>
      ) : isLoading ? (
        <span>Loading...</span>
      ) : (
        <span>Not ready ...</span>
      )}

      <div>{isFetching ? "Fetching..." : null}</div>
    </div>
  );
}
```

## Lazy Query

Use `enabled` to run a query only when certain conditions are met.

```tsx
useQuery({
  queryKey: ["todos", filter],
  queryFn: () => fetchTodos(filter),
  enabled: !!filter,
});
```

Perfect for:

- Fetching after the user selects a filter
- Fetching after entering a contact ID
- Search input queries

## `isLoading`

For lazy queries, prefer `isLoading` instead of `pending`:

- `isLoading` = **first-time fetching**
- Prevents showing a loading spinner before the query actually starts.

## `skipToken` **(TypeScript-only)**

A type-safe way to disable a query:

```tsx
queryFn: filter ? () => fetchTodos(filter) : skipToken;
```

> With `skipToken`, `refetch()` **will not work**.

## Summary

- `enabled: false` → fully paused query, can manually refetch.
- **Lazy query** → run only when needed (best practice for dependent-data).
- `skipToken` → safest disable method in TypeScript; no manual refetch.

**best practice pattern** for fetching:

- contacts → then addresses.
- parent → child data dependencies.
